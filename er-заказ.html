id [PK] INTEGER — автоинкремент
userId [FK] INTEGER, NULL — ссылка на User.id
status VARCHAR — pending / …
payload JSON — тело заказа (см. ниже)
createdAt TIMESTAMP

Структура поля payload (JSON) — пример содержимого заказа
{
  "userId": 1,
  "items": [
    {
      "id": 1,
      "name": "Chanel No.5",
      "price": 12000,
      "quantity": 2
    }
  ],
  "total": 24000,
  "paymentMethod": "card",
  "comment": "Доставка после 18:00"
}

При создании заказа (POST /api/orders) в payload сохраняется тело HTTP-запроса целиком; поле userId дублируется в отдельной колонке userId для связи с таблицей User.

Связь 1 : N: один пользователь может иметь несколько заказов; внешний ключ Order.userId → User.id; при удалении пользователя значение userId обнуляется (SetNull), запись заказа сохраняется.
