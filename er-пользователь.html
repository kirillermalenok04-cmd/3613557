ER-диаграмма сущности «Пользователь» (User)

ПОЛЬЗОВАТЕЛЬ (User)
id [PK] INTEGER — первичный ключ, автоинкремент
email VARCHAR (String) — уникальный, NOT NULL
passwordHash VARCHAR (String) — хеш пароля, NOT NULL
role VARCHAR (String) — роль: customer / admin, по умолчанию customer
createdAt TIMESTAMP (DateTime) — дата создания записи
updatedAt TIMESTAMP (DateTime) — дата последнего обновления
Связь с сущностью «Заказ» (Order):
1 пользователь → N заказов (1 : N)
Условные обозначения:
PK — первичный ключ (Primary Key);
типы полей соответствуют модели Prisma / СУБД PostgreSQL;
таблица в БД: User (схема public).
