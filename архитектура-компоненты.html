Основные потоки данных
Поток 1 Браузер запрашивает HTML, CSS, JS и отображает интерфейс магазина.
Поток 2 Клиент отправляет запросы к REST API (авторизация, заказы, отзывы) или использует localStorage для части сценариев.
Поток 3 Express обрабатывает запрос, Prisma формирует SQL-обращения к PostgreSQL.
Поток 4 Данные возвращаются в формате JSON клиенту; при деплое компоненты размещены на платформе Railway.

Рис. — Архитектурная диаграмма высокоуровневого взаимодействия компонентов системы: от клиентского браузера через сервер приложения и ORM к реляционной базе данных PostgreSQL
